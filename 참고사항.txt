설문지데이터 구조를 만들었고 그 데이터를 조회하는것 성공


*0. 설문작성 form을 이용하여 데이터베이스에 설문지 데이터를 전송시키기 (빈 form이 있으면 제거요청 메세지)
1. 설문지 데이터를 "DB조회"하여 진입아이템을 각 page화면에 그리기 ( home , search , 내정보 )
2. 진입아이템으로 진입후 설문지데이터로 설문지를 렌더링하여 응답할수있게 하기 ( 마감시간이 지나면 설문작성 X )
3. 설문 응답데이터를 "설문작성완료"를 누른뒤 DB저장 시키고 get으로 해당 설문지의 응답결과를 조회하여 차트 그려주기

당일 일정
설문응답 FORM을 마무리 아래는 통신값 구조 스케치
{ "0": ["주관식응답1","주관식응답2"]}   { "0": ["idx 0번객관식의 ㅁㅁ선택지", "일본", "제주도"], "1": ["idx 1번 객관식의 ㅁㅁ선택지", "어쿠스틱", "락"] }

주관식 인풋창에 onchange로 값을 갱신시켜 연동시키고 최종 통신때 전송시키기
이때 질문의 index를 가지고 배열의 원소로 만들어서 백엔드로 보낸뒤 백엔드에서 객체에 담는방식

객관식 각 질문마다 키 : [] 형태의 구조를 만든뒤 해당 질문키의 선택지에서 선택한 값을 키값 배열에 추가하고 제거하는 방식으로 값을 만든다
